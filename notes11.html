CSS 2: The Cascade

June 5, 2023

Odin Project Lesson

Specificity

Sometimes CSS styles will conflict and the way that the browser determines who wins is called the cascade.

Each selector has a value and those values determine their specificity in being applied over other styles.

    Some selectors and their values
  • Type: 1
  • Class: 10
  • ID: 100
  • In-line: 1000

This means that 101 type selectors would be required to override an ID applied style, or 11 class selectors (according to the video).

According to the lesson, specificity is not comparative - the ID selector will beat any number of class selectors.

The specificity of a rule is determined by the value of the element it applies to, as well as the number of selectors.

  • The Value of a selector is first to determine specificity. An ID specified rule will take precedence over class.
  • The number of selectors will also apply. A rule aimed at two classes will precede a rule aimed at a single class. A rule aimed at an ID and two classes will precede a rule aimed at the same ID and only one class.
  • The Universal Selector has no value, and any rules applying to all will only take effect if no other rules apply to an element.

Inheritance

If a style is applied to an element that contains another element, that style will usually be applied to the descendant elements.

Most typography based properties such as color and font will be inherited, while most other properties will not.

Inheritance can be easily overridden, as a style applied to a descendant element always takes precedence. Even if the parent is an ID and the child is class, the rule applying to the class will take precedence (unless the rule is a combinator applying specifically to the child)

Rule Order

The last aspect of the cascade determines which rule takes precedence if the specifictiy is the same.

When multiple styles are applied in a conflicting manner and are equally specific, whichever one is last (ie lower in the stylesheet) takes precedence.
